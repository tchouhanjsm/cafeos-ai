CaféOS Setup Guide

Fix the "Unexpected token '<'" error and other common issues

🔴 Common Error: "Unexpected token '<'… is not valid JSON"

What this means

The PHP API is returning HTML or XML instead of JSON. This happens when:

  • The database doesn't exist or MySQL isn't running
  • PHP's display_errors is On (XAMPP default) and an error leaks into the response
  • The cafeos folder is in the wrong location
  • The database was created with the wrong name

Step 1 — Run the diagnostic tool

Open this URL in your browser while XAMPP is running:

http://localhost/cafeos/backend/check.php

It will show you exactly which step is failing with instructions to fix it. Start here before anything else.

📦 Database Setup

⚠️ Common mistake: Wrong database name

The database must be named exactly cafeos — no underscore, no suffix.

If you created cafeos_db or cafe_os, you'll get a connection error.

Correct database setup

1
Open XAMPP Control Panel → Start Apache and MySQL
2
Open http://localhost/phpmyadmin
3
Click the SQL tab and run: 
CREATE DATABASE IF NOT EXISTS `cafeos` CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
4
Click cafeos in the left sidebar → click Import tab → choose file:
cafeos/backend/database/schema.sql → click Go
5
Verify: click cafeos in sidebar — you should see 8+ tables (staff, tables, orders, etc.)

Alternative: If your database is named differently

If you already have data in cafeos_db, edit one line in cafeos/backend/config/db.php:

private static string $dbName = 'cafeos_db';  // ← change this

🐘 PHP Error Display (XAMPP)

XAMPP ships with display_errors = On

This is fine for learning PHP, but it breaks API responses — any PHP warning gets printed as HTML before the JSON, making the whole response invalid.

Fix option A — .htaccess (already done in Phase 3 ZIP)

The file cafeos/backend/.htaccess should contain:

php_flag display_errors         Off
php_flag display_startup_errors Off
php_flag log_errors             On

If the file exists with these lines, Apache will override php.ini for this folder. Verify it's there.

Fix option B — php.ini

In XAMPP Control Panel → Apache → click Config → PHP (php.ini)

Find and change (around line 400):

display_errors = Off

Save → restart Apache in XAMPP Control Panel.

📁 File Location

The cafeos folder must be inside XAMPP's htdocs

Platform htdocs path Your files should be at
Windows C:\xampp\htdocs\ C:\xampp\htdocs\cafeos\
macOS /Applications/XAMPP/htdocs/ /Applications/XAMPP/htdocs/cafeos/
Linux /opt/lampp/htdocs/ /opt/lampp/htdocs/cafeos/
⚠️ Nested folder trap: If you extracted the ZIP inside another cafeos folder, you might have: htdocs/cafeos/cafeos/backend/ — that's wrong. It should be htdocs/cafeos/backend/

📦 npm Error: "Access token expired or revoked"

This means you have a stale npm auth token

Your .npmrc file has an expired token from a previous npm login. npm is trying to authenticate but the token is invalid.

Fix — clear the auth token (no account needed)

Open a terminal in the cafeos/electron/ folder and run:

# Option 1: Bypass auth entirely (use public registry directly)
npm install --registry https://registry.npmjs.org

# Option 2: Clear the stored token first, then install
npm logout
npm install

# Option 3: If behind a corporate proxy/firewall
npm config set registry https://registry.npmjs.org
npm install

You do not need an npm account to install packages. These are all public packages.

Windows: if "better-sqlite3" fails to compile

better-sqlite3 is a native module that needs C++ build tools:

  1. Download Visual Studio Build Tools
  2. During install, check "Desktop development with C++"
  3. After install, close and reopen terminal, then run npm install again

Or use the prebuilt binary:

npm install --ignore-scripts
npm rebuild better-sqlite3 --update-binary
